This is an unofficial, native, high-performance Java extension for Burp Suite that integrates the functionalities of Gitleaks into your passive scanning workflow, providing streamlined secret scanning. Built by a prowess security engineer, the extension offers extensive features that make secret scanning more efficient without external dependencies on binaries or Python wrappers.

Features

  • Native Montoya API: The extension is built on the modern Burp API ensuring future-proof compatibility.
  • Community Edition support: It detects Burp Suite Community Edition and registers a manual traffic handler, ensuring the scanning of traffic even without the Pro/Enterprise-only Passive Scanner.
  • Dedicated Issues tab: The extension offers an optional, experimental, standalone tab for managing and viewing findings separately from the main Burp Dashboard.
  • Safe verification: It has integrated context menu actions to verify secrets via Burp Repeater or step-by-step guides without automatic active scanning.
  • Base64 support: It automatically detects and recursively scans Base64 encoded payloads (up to depth 2).
  • Rule management: The tool enables you to auto-fetch the latest official Gitleaks rules, load custom or company-specific .toml configs, and disable specific rules via the User Interface.
  • Context menu: You can right-click any request in Burp to Force Rescan specific items.

Usage

The extension hooks into the Burps Passive Scan engine to conduct MIME Filtering, Aho-Corasick pre-scan, RE2J regex matching, entropy calculation, and Allowlisting.

graph TD
    A[HTTP Response] → B{MIME Filter}
    B -- Ignored (Images/Fonts) → Z[Stop]
    B -- Text/JSON/JS → C{Aho-Corasick Trie}
    C -- No Keywords Found → Z
    C -- Keywords Found → D[RE2J Regex Match]
    D → E{Entropy Check}
    E -- Low Entropy → Z
    E -- High Entropy → F[Allowlist Check]
    F -- Allowed → Z
    F -- Flagged → G[Burp Issue]

In the process:

  • MIME Filtering discards static assets (images, fonts, videos) to save cycles.
  • Aho-Corasick pre-scan battles with the response body checking for keywords from all active rules. The scan stops if no keywords wrap.
  • RE2J regex matching finds keywords and then runs the relevant linear-time regular expressions against the content.
  • Entropy calculation validates high-confidence matches using Shannon entropy to reduce false positives on random strings.
  • Allowlisting respects gitleaks:allow comments in-line and supports global allowlists defined in the toml configuration.

Moreover, the extension provides a safe, human-in-the-loop workflow to confirm if a detected secret is active without causing accidental account lockouts or unauthorized scanning.

  • Send to Repeater option for standard API keys, which builds a valid HTTP request and opens it in a new Repeater tab.
  • Copy verification cURL option that copies the equivalent curl command to your clipboard for a quick test.
